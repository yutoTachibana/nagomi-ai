# 安全性・倫理ガイドライン

このドキュメントは、こもれびの開発・運用に携わる全員が読むべき指針です。

## 1. 基本姿勢

こもれびを使う方の多くは、不安や絶望、孤独を抱えています。**コードの 1 行が誰かの夜を救うかもしれないし、悪化させるかもしれない**ことを忘れないでください。

## 2. クライシス検知の運用ルール

### 2.1 検知

`src/lib/safety/crisis-detector.ts` でキーワードマッチを行います。**機械学習による感情分析は採用しません**。理由:

- 誤判定が利用者を傷つけるリスクが大きい
- 「死にたい」と書いた人の文脈は機械では完全には読めない
- 説明可能でない判定は信頼を損なう

### 2.2 検知時の対応

1. AI の応答に共感の言葉を最初に置く
2. よりそいホットライン (0120-279-338) を 1 つだけ提示
3. UI 上にバナー表示（押し付けない）
4. `safety_events` テーブルに **type のみ** ログ（本文・キーワードは保存しない）

### 2.3 してはいけないこと

- 利用者の文章を Sentry / Datadog / 解析ツールに送る
- 「警察に通報します」のような圧力をかける
- 「あなたが死んだら悲しむ人が」という感情的束縛
- AI が自殺の方法・場所などについての具体的言及をする

## 3. 医療境界

ことね（AI）は次のことを **絶対に言いません**:

- 「あなたは ○○ 障害です」（診断）
- 「その薬は減らした方が」（投薬助言）
- 「すぐ良くなりますよ」（回復予言）
- 「もっと大変な人もいますよ」（相対化）
- 「がんばればなんとかなる」（精神論）

代わりに使う表現:

- 「専門医に相談する価値があるかもしれません」
- 「主治医に聞いてみてください」
- 「いまは本当にしんどい時期だと思います」
- 「あなたのつらさはあなたのものです」
- 「無理せず、できる範囲で」

## 4. データ最小主義

- 自由記述は **すべて端末で暗号化** してから保存（Phase 1）
- AI に送るデータは「会話に必要なものだけ」
- 解析・広告用の SDK を入れない
- ログには PII を含めない（メール・名前・本文）

## 5. 反エンゲージメント設計

利用者の依存・固着を促さない設計:

- 連続記録日数の表示・通知 → **入れない**
- 「もうすぐ達成」プッシュ通知 → **入れない**
- ログインボーナス → **入れない**
- DAU 最適化目的の機能 → **入れない**

代わりに肯定するもの:

- 「お休みする選択も大事です」
- 「書けない日があってもいいです」
- 「離れることも自分を大切にすることです」

## 6. 開発時のチェックリスト

PR を出す前に:

- [ ] 連続日数や DAU を可視化していないか
- [ ] AI に診断的な言葉を出させていないか
- [ ] エラー文言が利用者を責めていないか
- [ ] 自由記述をログ・解析に流していないか
- [ ] クライシス検知が動作するテストを書いたか
- [ ] アクセシビリティ (コントラスト・フォーカスリング) を確認したか

## 7. 連絡先

このガイドラインに反する実装、または懸念があれば即座に相談してください。
